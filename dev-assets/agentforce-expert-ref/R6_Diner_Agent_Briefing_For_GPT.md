## 1. Project Context

### What R6 Diner Is

R6 Diner is a **Digital Maître d'** Agentforce agent for a restaurant network. It handles:

- **Path 4 (Knowledge/RAG):** Policy, menu, directory questions via Prompt Templates + Data Library
- **Path 2 (Safety Net):** Human escalation with passcode auth gate
- **Path 1 (Transactional):** Modify/cancel reservations (stub — routes to escalation)
- **Path 3 (Concierge):** Recommendations (stub — uses menu/directory templates)

### Architecture

```
Agent Script (Orchestration)
    → Topic Selector (Intent_Classifier + EvaluateHardGates)
    → restaurant_information (Policy QA, Menu QA, Directory QA)
    → auth_required (passcode gate before escalation)
    → escalation (Escalation_Handler + DetermineEscalationPriority)
    → ambiguous_question (Ambiguous_Clarifier)
    → off_topic (static redirect)
```

**Data sources:**

- **Restaurant Lib** — File-based Data Library with 3 grounding files (no FAQ__kav)
- **Menu_Item__c** — Structured menu data via QueryMenuItems Apex
- **Account** — Restaurant metadata (Logic_State__c, Reliability_Score__c, etc.)
- **Contact** — Diner profile (allergies, membership, sentiment, passcode)

## 3. Next Steps (Manual in Salesforce UI)

### Step 2: Create and Configure the Agent

**Where:** Setup → Agents → New Agent (or open existing)

- **Label:** `R6 Diner Knowledge & FAQ Agent`
- **System Instructions:** Copy from `force-app/main/default/agentDefinitions/path4_knowledge_agent.agentScript` — the `system.instructions` block (lines 2–21)

**System instructions (copy-paste ready):**

```
You are the Digital Maitre d', the orchestration layer for the R6 Diner restaurant network.

ROLE: You coordinate between Prompt Templates, Actions, and Data Sources.
You NEVER answer diner questions directly. Every response is generated by a Prompt Template.
Your job is to:
  1. Gather context (diner profile, restaurant record, knowledge articles, menu data).
  2. Invoke the correct Prompt Template with the gathered context.
  3. Detect SYSTEM_FLAG_ESCALATE from QA templates and route to escalation. Execute transitions from Intent_Classifier.

ARCHITECTURE:
  Layer 1 — Orchestration: This agent script (topics, actions, transitions).
  Layer 2 — Reasoning: Prompt Templates (intent classification, policy QA, menu QA, directory QA, escalation, disambiguation).
  Layer 3 — Data: Data Library (Restaurant Lib), Menu_Item__c, Apex Actions, Native Get Record.

FINITE STATE MACHINE:
  Path 1 — Transactional (S4_Business_Gate): Modify/cancel reservations via ReservationManager.
  Path 2 — Safety Net (S3_Circuit_Breaker): Human escalation via Escalation_Handler template.
  Path 3 — Concierge (S2_Personalized_Grounding): Personalized recommendations.
  Path 4 — Knowledge (S5_Strict_RAG): Policy, menu, directory questions via QA templates.
```

**Note:** The org uses **Restaurant Lib** (file-based Data Library), not FAQ__kav. Policy/Directory/Ambiguous flows use grounding from the 3 files.

### Step 7: Wire Topics and Actions

**Where:** Agent Builder → Topics

| Topic | Type | Actions to Assign |

| **Ambiguous Question** | Clarifier | Clarify Question (Ambiguous_Clarifier), transition to topic_selector |
| **Off Topic** | Redirect | Transition to restaurant_information |
